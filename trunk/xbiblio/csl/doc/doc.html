Citation Style Language (CSL)

Basic Configuration

CSL is based on three types of structures. Fields represent data place-holders, and are combined in templates to configure output rendering. There are also rules for sorting, et al configuration and so forth.

Fields

Fields consist of two levels. The most general includes:

  • contributor
  • date
  • titles
  • locator
  • publisher

CSL then includes a host of more specific fields, which often inherit behavior from their parent field.

Reference Types and Fallbacks

Citation Class

Almost all citation styles can be reduced to a handful of more abstract classes:

author-date

Contains in-text citations that reference bibliographic entries. The reference list is sorted author-date, and duplicate keys are disambiguated using a year prefix. May contain additional locator details such as page number, and author may be supressed if it is duplicated in preceeding running text.

(Doe, 1999a, 1999b:34).

Doe, J. (1999a) Title, New York:ABC Publishers.

note

Citations are rendered as footnotes or endnotes, with optional bibliography. Where there is a bibliograph it is sorted author-date. This class typically includes positional rendering, with distinctions between full first and subsequent shortened forms, as well as conventions such as "ibid" for immediately subsequent duplicate citations.

1 Doe, J., 1999, Title.

Doe, J. (1999a) Title, New York:ABC Publishers.

number

Citations are in-text numbered labels, and bibliographies may be ordered either by the order of their approach as cited, or author-date.

[1]

[1] Doe, J., 1999, Title, New York:ABC Publishers.

label

Citations are in-text natural language labels, and bibliographies may be ordered either by that label, or author-date.

[doe99]

[doe99] Doe, J., 1999, Title, New York:ABC Publishers.

annotated

There are no citations per se, and reference entries includes annotation text.

Doe, J., 1999, Title, New York:ABC Publishers.

Some notes ...

CSL is designed to make configuration of these classes simple, but also to allow for custom definitions that override the default assumptions of the base classes. For example, to implement the author-date class using the explicit definitions of the custom definition, one would first define default author rendering and substitution like so:

  <defaults>
    <contributor name-as-sort-order="no">
      <name and="symbol" initialize-with=". " delimiter=", " delimiter-precedes-last="always"/>
      <label form="short" prefix=", " text-transform="capitalize" suffix="."/>
    </contributor>

    <author name-as-sort-order="all">
      <name and="symbol" sort-separator=", " initialize-with="." delimiter=", " delimiter-precedes-last="always"/>
      <label form="short" prefix=" (" suffix=".)" text-transform="capitalize"/>
      <substitute>
        <choose>
          <editor/>
          <translator/>
          <titles/>
        </choose>
      </substitute>
    </author>

    <locator>
      <number/>
    </locator>

    <identifier>
      <number/>
    </identifier>

    <titles>
      <title/>
    </titles>

    <date>
      <year/>
      <month prefix=", "/>
      <day prefix=" "/>
    </date>

    <publisher>
      <place suffix=": "/>
      <name/>
    </publisher>

    <access>
      <text term-name="retrieved" text-transform="capitalize"/>
      <date suffix=", ">
        <month suffix=" "/>
        <day suffix=", "/>
        <year/>
      </date>
      <text term-name="from"/>
      <url/>
    </access>

  </defaults>
